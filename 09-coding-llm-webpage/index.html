Key Takeaways

What matters for on-device coding generation with the GX10.

Qwen3-Coder dominates at 71 tok/s

At 30B parameters, Qwen3-Coder is over 5x faster than Devstral and 3x faster than DeepCoder on the GB10 GPU. It also produces the richest output at 550+ lines and ~18 KB, completing the task in just over a minute.

DeepCoder is the efficiency pick

At 14B / 9 GB VRAM, DeepCoder uses the least memory while still delivering valid, working outputs. It achieves a solid 22 tok/s, making it a strong choice when running alongside other models or services.

Devstral trades speed for canvas-based output

Devstral's 14 tok/s is the slowest of the three, but 2 of its 3 runs produced canvas-based rendering (as opposed to pure CSS/HTML), showing a different approach to the same problem. The tradeoff is a 3.5-minute generation time.

All three produce valid, working code

Every run from every model produced structurally valid HTML with proper DOCTYPE and closing tags. All outputs are interactive and renderable in a browser, demonstrating that local coding LLMs on the GX10 are production-viable for single-file tasks.

Warm TTFT under 200ms across the board

Once loaded, all three models begin generating in under 200 milliseconds. Cold start ranges from 4.3s (Qwen3-Coder) to 10.9s (Devstral), reflecting model size and Ollama load time.

122 GB unified memory = room for all three

The combined VRAM footprint of all three models is 41 GB, well within the GX10's 122 GB unified memory. You could run all three simultaneously with plenty of headroom for embeddings, rerankers, and other services.

Prompt Used

The exact same prompt was sent to all three models with no system prompt.

Benchmark Prompt prompt.txt
Build a single HTML file (no external dependencies) that creates an interactive 3D solar system visualization. Requirements: 1. SOLAR SYSTEM: Sun at center with at least 8 planets orbiting at different speeds and distances. Each planet should have accurate relative sizes and colors (Mercury gray, Venus yellow, Earth blue-green, Mars red, Jupiter orange-striped, Saturn with rings, Uranus cyan, Neptune blue). 2. 3D EFFECT: Use CSS 3D transforms or canvas to create a perspective view of the orbital plane. The system should look three-dimensional, not flat. 3. ANIMATIONS: Planets must orbit the sun continuously with smooth animation. Inner planets faster, outer planets slower. Include a subtle rotation on the sun (glowing effect). 4. INTERACTIVITY: - Click any planet to show an info card with the planet name, distance from sun, orbital period, and a fun fact - Hover over planets to highlight them and show the name - Add a speed control slider (0.5x to 5x) - Add a pause/play button - Add a toggle between top-down view and perspective view 5. VISUAL POLISH: - Dark space background with twinkling stars (animated) - Orbital path lines (subtle, dashed) - Glow effect on the sun - Smooth transitions on all interactions - Planet shadows/lighting effect 6. RESPONSIVE: Must work on both desktop and mobile screens. 7. UI PANEL: A sleek side panel showing all planets as a list, clicking any name should highlight that planet and show its info. Output ONLY the complete HTML file content, starting with <!DOCTYPE html> and ending with </html>. No explanations before or after.
